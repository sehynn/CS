# Long Tail & Power Law

## 1. Long Tail

> “상위 몇 개만 잘 팔리는 게 아니라,
> 안 유명한 수많은 것들의 합도 엄청 크다.”

### 대표 예시

* Netflix 영화
* YouTube 영상
* Amazon 상품

 
### 기존 오프라인 구조

오프라인 매장은 진열 공간이 제한적이기 때문에
주로 베스트셀러 위주로 판매했다.

```text
[ 베스트셀러 중심 ]
``` 

### 인터넷 서비스의 등장

온라인 플랫폼은:

* 재고 비용이 낮고
* 진열 공간 제약이 거의 없으며
* 마이너 상품도 계속 제공 가능하다.

즉:

* 인기 상품 → Head
* 덜 유명하지만 수가 많은 상품 → Tail

이 긴 Tail 구간의 총합이 커지는 현상을 **Long Tail** 이라고 한다.
 

### 간단한 분포 형태

```text
판매량
 ^
 |■■■■■■■■■■
 |■■■■
 |■■
 |■
 |■
 |■
 +-------------------->
   인기순위
```

뒤로 긴 꼬리(Tail)가 이어지는 형태.
 

### 핵심 특징

* 소수 인기 콘텐츠만 중요한 것이 아님
* 다수의 마이너 콘텐츠 합도 매우 큼
* 온라인 플랫폼에서 특히 강하게 나타남

---

## 2. Power Law (멱함수)

어떤 값이 커질수록 발생 확률이
특정 비율로 감소하는 분포.

### 공식

```math
P(x) ∝ x^-α
```
 
### 의미

* 작은 값은 매우 많음
* 큰 값은 매우 적음
* 하지만 큰 값도 예상보다 자주 등장함

즉:

> “극소수가 대부분을 차지하는 구조”
 

### 대표 예시

* 부의 분포
* SNS 팔로워 수
* GitHub 스타 수
* 도시 인구
* 조회수
* API 트래픽

예:

* 대부분 유튜버는 조회수가 낮음
* 극소수 유튜버만 수백만 조회수를 가짐
* 하지만 전체 트래픽 대부분은 그 소수가 차지

---

### 간단한 형태

```text
빈도
 ^
 |■■■■■■■■■■■■■■
 |■■■■■■
 |■■■
 |■
 |■
 +-------------------->
      크기
```
 

## 3. Long Tail vs Power Law

| 구분        | Long Tail                         | Power Law                |
| --------- | --------------------------------- | ------------------------ |
| 관점        | 비즈니스/서비스 현상                       | 수학적 분포                   |
| 핵심        | 마이너한 것들의 총합이 큼                    | 극소수가 대부분 차지              |
| 자주 등장하는 곳 | 콘텐츠 플랫폼                           | 자연/사회 데이터 전체             |
| 관계        | Long Tail 현상은 종종 Power Law 분포를 따름 | Long Tail의 수학적 기반이 되기도 함 |
  

## 4. 실제 서비스에서 중요한 이유

현실 서비스의 데이터는 대부분 균등하지 않다.

예:

* 소수 유저가 대부분의 트래픽 생성
* 소수 API가 대부분 호출됨
* 특정 게시글만 폭발적으로 조회됨

그래서 아래 기술들이 중요해진다:

* Cache
* CDN
* Rate Limiting
* Hot Key 처리
* Sharding
* Message Queue
 
## 5. 한 줄 요약

* Long Tail
  → “안 유명한 수많은 것들의 합도 크다”

* Power Law
  → “극소수가 대부분을 차지한다”

* 관계
  → Long Tail 현상은 Power Law 형태로 나타나는 경우가 많다.
